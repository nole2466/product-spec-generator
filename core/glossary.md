# 術語表（Glossary）

> 統一用語，避免溝通歧義

---

## 角色術語

| 術語 | 全稱 | 說明 |
|-----|------|------|
| PM | Product Manager | 產品經理，定義需求 |
| PD | Product Designer | 產品設計師，需求轉規格 |
| Backend | Backend Developer | 後端工程師 |
| Frontend | Frontend Developer | 前端工程師 |
| QA | Quality Assurance | 品質保證工程師 |

---

## 文件術語

| 術語 | 說明 |
|-----|------|
| spec.md | 功能規格書，描述功能的 What |
| design.md | 設計規格書，描述 UI/UX |
| contract.md | API 契約，前後端資料約定 |
| acceptance.md | 驗收標準，測試情境 |
| mockup | 設計稿、線框圖 |

---

## 狀態術語

| 術語 | 說明 | 可編輯 |
|-----|------|:------:|
| draft | 草稿，撰寫中 | ✅ |
| review | 審核中 | ⚠️ 僅小修 |
| approved | 已核准，可開發 | ❌ |
| done | 已完成，已上線 | ❌ |

---

## 流程術語

| 術語 | 說明 |
|-----|------|
| Spec Review | 規格審核會議 |
| In Scope | 本次要做的範圍 |
| Out of Scope | 本次不做的範圍 |
| Edge Case | 邊界情境 |
| Happy Path | 正常流程路徑 |

---

## 技術術語（僅用於 contract.md）

| 術語 | 說明 |
|-----|------|
| Endpoint | API 端點 |
| Request | API 請求 |
| Response | API 回應 |
| Schema | 資料結構定義 |
| Query | URL 查詢參數 |
| Body | 請求主體 |

---

## Level 術語

| 術語 | 工時 | 團隊規模 |
|-----|-----|---------|
| Level 0 | < 1 天 | 1 人 |
| Level 1 | 1-5 天 | 1-2 人 |
| Level 2 | 1-2 週 | 3-5 人 |
| Level 3 | > 2 週 | > 5 人 |
